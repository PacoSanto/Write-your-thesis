# Contributing

Thank you for considering contributing to this template!

## How to contribute

1. **Fork** the repository.
2. **Create a branch** for your changes: `git checkout -b feature/your-improvement`.
3. **Make your changes** and test that the template still compiles without errors.
4. **Commit** with a clear message: `git commit -m "Add support for XYZ"`.
5. **Push** to your fork and open a **Pull Request**.

## What kind of contributions are welcome?

- Bug fixes (compilation errors, broken references, typos)
- New useful packages or commands with clear documentation
- Improvements to the README or cheat sheets
- Translations of the documentation
- Compatibility fixes for different LaTeX distributions or Overleaf
- Examples of common thesis elements (complex tables, multi-panel figures, etc.)

## Guidelines

- Keep comments in English so the template remains accessible internationally.
- Every new package or command should include a short comment explaining what it does and how to use it.
- Test your changes by compiling the full document (`latexmk maintext_thesis.tex`) before submitting.
- Do not commit PDF output files or LaTeX auxiliary files (they are in `.gitignore`).

## Reporting issues

If you find a problem but don't have time to fix it yourself, please open an **Issue** on GitHub with:
- A description of the problem
- Your LaTeX distribution and version
- The error message (if any)

Thank you!
